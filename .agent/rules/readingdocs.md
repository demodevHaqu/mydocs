### Documentation Rules and Guidelines of Reading for Docs

**\*** Core Principles **\***

1. Clarify First

- Use technically accurate yet easy-to-understand language
- Define technical terms upon first use
- Break down complex concepts step by step

2. Visual Enhancement

- Include diagrams or screenshots for all major concepts
- Format code examples with visual distinction
- Actively utilize flowcharts and architecture diagrams

3. Completeness

- Clearly state prerequisites
- Document every step without omissions
- Include expected results and troubleshooting methods
