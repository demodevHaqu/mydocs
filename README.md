### Google Antigravity 문서 ###
본 문서는 공식 Google Antigravity 문서의 핵심 개념, 기능 및 사용법을 다루는 포괄적인 요약입니다.
1. 소개
Google Antigravity는 전통적인 IDE를 에이전트 우선 경험으로 진화시킨 에이전트 개발 플랫폼입니다. 개발자가 친숙한 AI IDE 핵심을 유지하면서 워크스페이스 전반에 걸쳐 에이전트를 관리하여 더 높은 작업 지향적 수준에서 작업할 수 있도록 합니다.
주요 기능:

에이전트 워크플로우: 에이전트를 독립적인 표면으로 추출하여 에디터, 터미널 및 브라우저 전반에 걸쳐 자율적으로 작동할 수 있는 도구를 제공합니다.
작업 지향적: 작업 및 아티팩트를 통한 검증과 상위 수준 커뮤니케이션을 강조합니다.
엔드투엔드 개발: 에이전트가 기능 구축 및 UI 반복에서 버그 수정 및 연구에 이르기까지 복잡한 소프트웨어 작업을 계획하고 실행할 수 있습니다.

주요 기능

AI 기반 IDE: Agent, Tab, Command와 같은 기능을 포함합니다.
비동기 에이전트: 로컬 에이전트가 워크스페이스에서 병렬로 작업합니다.
에이전트 매니저: 계획, 대화 및 아티팩트 검토를 위한 전용 뷰입니다.
멀티 윈도우: Editor, Manager 및 Browser 윈도우를 지원합니다.


2. 핵심 개념
에이전트
에이전트는 Antigravity의 핵심 AI 기능입니다. 다음과 같은 작업을 수행할 수 있는 최첨단 LLM 기반의 다단계 추론 시스템입니다:

기존 코드에 대한 추론
다양한 도구 사용(브라우저 포함)
작업, 아티팩트 등을 통해 사용자와 소통

핵심 구성 요소:

추론 모델: 에이전트를 구동하는 기본 LLM
도구: 에이전트가 사용할 수 있는 기능(파일 편집, 터미널, 브라우저 등)
아티팩트: 에이전트가 생성한 출력물(계획, 코드, 다이어그램)
지식: 인사이트의 영구 메모리
커스터마이제이션: 사용자 정의 설정 및 규칙

모델
Antigravity는 다양한 추론 모델을 지원하여 개발자가 작업에 가장 적합한 모델을 선택할 수 있습니다:

Gemini 3 Pro
Claude Sonnet 4.5
GPT-OSS

에이전트 모드 및 설정
에이전트는 다양한 모드로 작동할 수 있습니다:

플래닝 모드: 구조화된 접근 방식이 필요한 복잡한 작업용
빠른 모드: 빠르고 작은 작업용

설정에는 사용자가 에이전트 출력물을 검토하는 방법과 시기를 제어하는 아티팩트 검토 정책과 같은 옵션이 포함됩니다.
작업 그룹
플래닝 모드에서 에이전트는 작업 그룹을 사용하여 크고 복잡한 작업을 처리합니다.

분해: 문제를 더 작고 접근 가능한 작업 단위로 나눕니다.
병렬 처리: 에이전트가 작업의 여러 부분을 동시에 처리할 수 있습니다.
프레젠테이션: 작업 섹션은 목표 요약과 빠른 감사를 위한 편집된 파일 목록을 포함하여 사용자에게 변경 사항을 제시합니다.

지식
지식 항목은 영구 메모리 시스템 역할을 합니다. 코딩 세션의 인사이트, 솔루션 및 컨텍스트를 캡처하여 에이전트 매니저에서 저장하고 볼 수 있습니다. 이를 통해 에이전트가 이전 작업에서 "학습"할 수 있습니다.

3. 에디터 경험
에디터는 VS Code를 기반으로 하지만 풍부한 AI 기능으로 향상된 기본 진입점입니다.
에디터 개요

친숙함: 표준 에디터처럼 느껴지도록 설계되었습니다(파일 열기, 탭 전환, 직접 편집).
AI 통합: Agent, Tab 및 Command 기능과 깊이 통합되어 있습니다.
확장 프로그램: 구문 강조, 소스 제어 등을 위해 Open VSX 마켓플레이스 확장 프로그램을 지원합니다.

Tab 및 내비게이션

Supercomplete: 커서 근처에 코드 제안을 제공합니다. 파일 전체에서 코드를 수정할 수 있습니다(예: 변수 이름 변경, 함수 정의 업데이트).
Tab-to-Jump: 커서를 이동할 다음 논리적 위치를 제안하는 유동적인 내비게이션 도구입니다.
Tab-to-Import: (암시적) 모듈 가져오기를 돕습니다.

Command
Command는 워크플로우에 자연어를 도입합니다.

트리거: Command + I(Mac) 또는 Ctrl + I(Windows/Linux)
사용법: 자연어로 요청을 입력합니다(예: "이 목록을 정렬하는 함수 만들기").
실행: Antigravity가 검토를 위해 코드 또는 명령을 인라인으로 생성합니다.

에이전트 사이드 패널
에디터 오른쪽에 위치합니다.

기능: 에이전트와 직접 작업합니다.
기능: 새 대화 시작, 이미지 첨부, 에이전트 모드 전환, 모델 선택.
추적: 하단 도구 모음을 통해 열린 파일 변경, 실행 중인 터미널 프로세스 및 아티팩트를 추적합니다.

변경 사항 검토 + 소스 제어

변경 사항 검토: 에이전트 패널의 하단 도구 모음에 있는 섹션을 통해 대화에서 에이전트와 사용자가 수행한 모든 변경 사항을 스크롤할 수 있습니다.
상호 작용: 사용자는 파일 diff에 댓글을 달아 에이전트에게 피드백을 전달할 수 있습니다.


4. 기능 및 도구
브라우저 서브에이전트
에이전트가 웹과 상호 작용해야 할 때 브라우저 서브에이전트를 사용합니다.

전문 모델: 브라우저 작업에 특화된 모델을 실행합니다.
기능: 클릭, 스크롤, 타이핑, 콘솔 로그 읽기, DOM 캡처, 스크린샷, 마크다운 파싱 및 비디오 녹화.
오버레이: 페이지를 제어할 때 파란색 테두리와 작업 패널을 표시합니다.
백그라운드 작업: 포커스되지 않은 탭에서 작업할 수 있어 사용자가 다른 탭에서 계속 작업할 수 있습니다.

MCP(Model Context Protocol)
Antigravity는 MCP와 통합되어 에디터가 로컬 도구 및 외부 서비스에 연결할 수 있습니다. 이를 통해 에이전트에게 즉각적인 코드베이스를 넘어서는 실시간 컨텍스트와 기능을 제공합니다.

5. 아티팩트
아티팩트는 에이전트가 작업을 수행하거나 소통하기 위해 생성하는 유형적 출력물입니다.
아티팩트 유형

작업 목록: 복잡한 작업의 진행 상황을 모니터링하는 데 사용됩니다.
구현 계획: 사용자 검토 및 댓글을 위해 코드 변경을 설계합니다.
워크스루: 스크린샷 및 녹화를 포함하여 수행된 변경 사항을 요약합니다.
스크린샷: 브라우저 서브에이전트가 캡처한 이미지.
브라우저 녹화: 브라우저 서브에이전트 작업의 비디오 녹화.
다이어그램/코드: (암시적) 기타 생성된 콘텐츠.


6. 에이전트 매니저 및 워크스페이스
에이전트 매니저
에이전트 작업을 감독하기 위한 상위 수준 뷰입니다.

조감도: 여러 워크스페이스와 수십 개의 에이전트를 동시에 관리합니다.
상호 작용: 주로 에이전트를 통해 코드베이스와 상호 작용합니다.
내비게이션: CMD+E / CTRL+E로 에디터와 매니저 간에 전환합니다.
윈도우 관리: 매니저에서 에디터 윈도우를 숨기거나 포커스하거나 닫습니다.

워크스페이스

멀티 워크스페이스: 에이전트 매니저에서 여러 워크스페이스를 동시에 작업합니다.
생성: 시작 폴더를 선택하여 새 워크스페이스를 엽니다.
전환: 사이드바를 통해 워크스페이스 간의 대화를 전환합니다.
